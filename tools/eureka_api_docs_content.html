Date Filter Tutorial

Eureka-Search API supports the following date filters to filter documents by date:

Supported Fields:

  • created_date

  • modified_date

  • indexed_date

Usage Examples:

  1. Filter by a specific date:

    "modified_date": ["2024-11-26T00:00:00Z TO 2024-11-26T23:59:59Z"]

  2. From first date to a specific date:

    "modified_date": ["* TO 2024-11-26T23:59:59Z"]

  3. From a specific date to latest date:

    "modified_date": ["2024-11-20T00:00:00Z TO *"]

  4. From a specific date to now:

    "modified_date":["NOW-1DAY TO NOW"]
    "modified_date": ["2024-11-01T00:00:00Z TO NOW"]

  5. From a specific date to a specific date with added days:

    "modified_date": ["2024-11-01T00:00:00Z TO 2024-11-01T00:00:00Z+5DAY"]

Query Examples with {“action”: “get_source_info”}

:info:atlassian-info#F4F5F7

The function { "action": "get_source_info" } allows users to retrieve source information for fields and their values.

The returned fields and values are strings, and they are used for the "field" and "filter" functions. Since text fields are analyzed by word, Eureka does not return their values. Only string fields—such as id, product, version, etc. included.

If you need all fields to reference for raw_query, you can use "field": ["all_fields"]. This will return all fields, including text fields, for search purposes.
